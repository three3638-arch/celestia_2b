# Celestia Jewelry - B2B 外贸电商平台业务蓝图

> 版本: v1.2 | 最后更新: 2026-03-28 | 状态: **已确认**

---

## 一、项目概述

**Celestia Jewelry** 是一个面向中东（沙特阿拉伯为主）B2B 批发客户的银首饰在线订货平台。

- **业务模式**: B2B 询价制，商品展示参考价（SAR 沙特里亚尔），以订单报价为准
- **主营品类**: 925银首饰（莫桑石 / 锆石 镶嵌）
- **核心流程**: 客户浏览选品 → 提交订单 → 卖家报价（CNY）→ 确认汇率换算（SAR）→ 协商调整 → 确认付款 → 发货物流
- **货币体系**: 管理端操作币种为 **人民币（CNY）**，客户端展示币种为 **沙特里亚尔（SAR）**，每次订单确认报价时锁定当次汇率
- **品牌风格**: 黑底金字，高端优雅（参考已有 Logo 设计）

## 二、用户角色与权限

### 2.1 管理员（卖家 / 平台运营方）

| 权限 | 说明 |
|------|------|
| 商品管理 | 上传 Excel 导入商品、编辑 / 上下架 / 删除商品 |
| 客户管理 | 审核客户注册、查看客户列表与详情 |
| 订单报价 | 为客户订单中每个 SKU 填写单价、标记缺货 |
| 订单管理 | 更新订单状态、确认收款、录入物流信息 |
| 数据概览 | 仪表盘查看订单/商品/客户统计数据 |

### 2.2 客户（B2B 批发买家）

| 权限 | 前提条件 |
|------|---------|
| 注册账号 | 填写姓名和电话号码即可 |
| 浏览商品 | 需注册并通过管理员审核 |
| 加入购物车 / 提交订单 | 需审核通过 |
| 查看报价 / 修改订单 | 订单在协商阶段时 |
| 确认订单 | 对报价满意后 |
| 查看物流 / 历史订单 | 随时可查 |

### 2.3 客户准入流程

```
客户注册（填写姓名、电话号码）
        ↓
   状态：待审核（可登录但无法查看商品）
        ↓
   管理员审核通过（业务员会联系客户）
   审核时必须为客户设置加价比例（默认预填 1.15，即加价 15%）
        ↓
   状态：已激活（可正常使用全部功能）
```

## 三、商品模型（SPU / SKU）

### 3.1 品类结构

```
首饰品类
├── 戒指 (Ring)
├── 项链 (Necklace)
├── 手链 / 手镯 (Bracelet / Bangle)
├── 耳钉 / 耳环 (Earring)
└── 其他 (Other)
```

### 3.2 SPU（款式）

SPU 代表一个首饰款式设计，包含以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| SPU编号 | 唯一标识 | `CJ-NK-001` |
| 名称（中文） | 原始录入名 | 星辰吊坠项链 |
| 名称（英文） | AI翻译 | Starlight Pendant Necklace |
| 名称（阿拉伯文） | AI翻译 | قلادة بدلاية ستارلايت |
| 品类 | 分类归属 | 项链 |
| 描述（中/英/阿） | 商品详细介绍 | ... |
| 图片 | **多张**，SPU级别共享（首图 + 其他图片） | 首图 + 细节图 |
| 参考价区间(SAR) | 最低价~最高价，均可为空，仅供参考，展示时标注"价格随市场波动，以订单报价为准" | SAR 80~200 |
| 可用宝石类型 | 该款式支持的宝石 | 莫桑石 / 锆石 / 两者都有 |
| 状态 | 上架 / 下架 | 上架 |

**参考价展示规则（客户端）：**

- 最低价和最高价都有：`SAR 80~200`
- 只有最低价：`SAR 80 起`
- 只有最高价：`最高 SAR 200`
- 都为空：`价格面议`
- 客户实际看到的参考价 = SPU 参考价 × 该客户的加价比例（见 5.4 节）

### 3.3 SKU（规格）

SKU 是 SPU 下的具体可售单元，由以下属性维度组合生成：

| 属性维度 | 适用品类 | 可选值示例 |
|---------|---------|-----------|
| **宝石类型** | 全部 | 莫桑石(Moissanite) / 锆石(Zircon) |
| **金属底色** | 全部 | 银色(Silver) / 金色(Gold) / 玫瑰金(Rose Gold) / 其他(Other) |
| **戒指尺码** | 仅戒指 | 5 / 6 / 7 / 8 / 9 / 10 ... |
| **链长度** | 项链、手链 | 40cm / 45cm / 50cm / 16cm / 18cm ... |

> 注意：不是所有维度都适用于所有品类。耳钉只有 宝石类型 × 金属底色。SKU 没有独立图片，图片统一挂在 SPU 级别。

**SKU 生成示例：**

```
SPU: 星辰吊坠项链 (CJ-NK-001)
  ├── CJ-NK-001-MO-SIL-45: 莫桑石 + 银色 + 45cm
  ├── CJ-NK-001-MO-GLD-45: 莫桑石 + 金色 + 45cm
  ├── CJ-NK-001-MO-SIL-50: 莫桑石 + 银色 + 50cm
  ├── CJ-NK-001-ZR-SIL-45: 锆石 + 银色 + 45cm
  ├── CJ-NK-001-ZR-GLD-45: 锆石 + 金色 + 45cm
  └── CJ-NK-001-ZR-SIL-50: 锆石 + 银色 + 50cm

SPU: 永恒之环戒指 (CJ-RG-002)
  ├── CJ-RG-002-MO-SIL-6: 莫桑石 + 银色 + 尺码6
  ├── CJ-RG-002-MO-SIL-7: 莫桑石 + 银色 + 尺码7
  ├── CJ-RG-002-MO-GLD-6: 莫桑石 + 金色 + 尺码6
  └── ...

SKU编号规则：{SPU编号}-{宝石缩写}-{金属缩写}-{尺码或长度}
  宝石缩写：MO=莫桑石, ZR=锆石
  金属缩写：SIL=银色, GLD=金色, RSG=玫瑰金, OTH=其他
```

**SKU 字段：**

| 字段 | 说明 |
|------|------|
| SKU编号 | 按规则自动生成（可读格式），支持手动覆盖 |
| 所属SPU | 关联的款式 |
| 宝石类型 | 莫桑石 / 锆石 |
| 金属底色 | 银色 / 金色 / 玫瑰金 / 其他 |
| 尺码/长度 | 按品类不同，可为空 |
| 参考价区间(SAR) | 展示用参考价格区间（最低价/最高价），沙特里亚尔，均可为空 |
| 库存状态 | 有货 / 缺货 / 预订 |

> SKU 没有独立图片字段。

### 3.4 图片说明

- 图片挂在 **SPU 级别**，不为每个 SKU 单独设图
- 每个 SPU 有一张**首图**（必需）和多张**其他图片**（可选）
- 图片来源：Excel 中嵌入，导入时自动提取并上传到云存储

## 四、Excel 导入模板规范

### 4.1 设计原则

- 用户从各厂家获得不同格式的 Excel，**手动整理成统一模板格式**后上传
- 上传时系统自动：提取嵌入图片、调用 AI 翻译中文名称/描述为英文和阿拉伯文
- 模板采用 **单 Sheet** 结构，每行一个 SPU，SKU 属性通过逗号分隔在单元格内编码，系统自动展开为多个 SKU

### 4.2 模板列定义（单 Sheet）

| 列 | 字段名 | 必填 | 说明 | 示例 |
|---|--------|------|------|------|
| A | SPU编号 | 是 | 唯一标识 | `CJ-NK-001` |
| B | 首图 | 是 | 嵌入在单元格中的主图（1张） | (嵌入图片) |
| C | 名称 | 否 | 中文名称（为空时以SPU编号代替展示） | 星辰吊坠项链 |
| D | 品类 | 是 | 戒指/项链/手链/耳钉/其他 | 项链 |
| E | 宝石类型 | 是 | 逗号分隔 | `莫桑石,锆石` |
| F | 金属底色 | 是 | 逗号分隔 | `银色,金色` |
| G | 尺码 | 否 | 戒指用，逗号分隔 | `6,7,8,9` |
| H | 长度(cm) | 否 | 项链/手链用，逗号分隔 | `40,45,50` |
| I | 参考价最低(SAR) | 否 | 价格区间下限 | `80` |
| J | 参考价最高(SAR) | 否 | 价格区间上限 | `200` |
| K | 描述 | 否 | 中文描述 | 925银镶嵌莫桑石吊坠... |
| L+ | 其他图片 | 否 | 附加图片，可多列多张 | (嵌入图片) |

### 4.3 SKU 自动展开规则

系统读取每行 SPU 的逗号分隔属性，自动做**笛卡尔积**展开生成 SKU：

**示例：一行 Excel 数据**

| SPU编号 | 首图 | 品类 | 宝石类型 | 金属底色 | 长度 | 参考价最低 | 参考价最高 |
|---------|------|------|---------|---------|------|-----------|-----------|
| CJ-NK-001 | (图) | 项链 | 莫桑石,锆石 | 银色,金色 | 45,50 | 80 | 200 |

**自动展开为 2x2x2 = 8 个 SKU：**

| SKU编号 | 宝石 | 金属 | 长度 |
|---------|------|------|------|
| CJ-NK-001-MO-SIL-45 | 莫桑石 | 银色 | 45cm |
| CJ-NK-001-MO-SIL-50 | 莫桑石 | 银色 | 50cm |
| CJ-NK-001-MO-GLD-45 | 莫桑石 | 金色 | 45cm |
| CJ-NK-001-MO-GLD-50 | 莫桑石 | 金色 | 50cm |
| CJ-NK-001-ZR-SIL-45 | 锆石 | 银色 | 45cm |
| CJ-NK-001-ZR-SIL-50 | 锆石 | 银色 | 50cm |
| CJ-NK-001-ZR-GLD-45 | 锆石 | 金色 | 45cm |
| CJ-NK-001-ZR-GLD-50 | 锆石 | 金色 | 50cm |

> 如果某个属性列为空（如耳钉没有尺码和长度），则该维度不参与组合。

### 4.4 导入流程

```
上传 Excel 文件
      ↓
系统解析单 Sheet：逐行读取 SPU，按逗号分隔属性做笛卡尔积展开 SKU
      ↓
自动提取嵌入图片 → 上传到云存储（Cloudflare R2）
      ↓
调用 AI 翻译：中文名称/描述 → 英文 + 阿拉伯文
      ↓
预览页面：抽样展示解析结果（前 N 个 SPU 及其 SKU + 图片 + 翻译预览）
      ↓
显示导入摘要：共 X 个 SPU、Y 个 SKU、Z 张图片
      ↓
用户确认 → 写入数据库
```

> 由于单次导入数据量可能较大（数千商品），预览页面采用**抽样展示**策略：展示前 5~10 个 SPU 的完整信息作为样本，同时显示整体统计摘要（总SPU数、总SKU数、总图片数），供用户快速确认数据是否正确。

## 五、订单生命周期（非线性流转）

### 5.1 订单状态定义

| 状态 | 英文标识 | 说明 | 可操作方 |
|------|---------|------|---------|
| 待报价 | `pending_quote` | 客户已提交，等待卖家报价 | 管理员 |
| 已报价 | `quoted` | 卖家已完成报价（含汇率+加价换算），等待客户反馈 | 客户 |
| 协商中 | `negotiating` | 客户修改了订单内容，等待卖家重新报价 | 管理员 |
| 已确认 | `confirmed` | 双方对订单内容和价格达成一致，等待付款 | - |
| 部分付款 | `partially_paid` | 客户已付定金，尾款待付 | 管理员确认 |
| 已付清 | `fully_paid` | 全部款项已到账 | 管理员 |
| 已发货 | `shipped` | 已填写物流单号（可在部分付款后即发货） | 客户查看 |
| 结算中 | `settling` | 货物已到，双方线下对账/退货/调整中，管理员可修改订单项 | 管理员 |
| 已完成 | `completed` | 订单最终结算完毕 | - |
| 已取消 | `cancelled` | 订单取消（见取消规则） | 见下方说明 |

> **关键规则**：
> - **发货**：不要求全额付清。管理员可在「部分付款」状态下即发货（如客户付定金后发货，到货后付尾款）。
> - **结算**：订单满足（已发货 + 已付清）后，进入「结算中」环节，管理员可根据到货实际情况调整订单（退货、数量修改、折扣），最终确认结算完成。
> - **取消规则**：未产生任何付款记录时，客户或管理员均可取消；**已产生付款记录后，仅管理员可取消**（需线下处理退款）。

### 5.2 订单项状态

每个订单中的商品（订单项）有独立状态：

| 状态 | 说明 |
|------|------|
| 待报价 | 等待卖家填写价格 |
| 已报价 | 卖家已填价格 |
| 缺货 | 卖家标记为缺货 |
| 客户移除 | 客户因价格/预算移除 |
| 已确认 | 双方确认此商品进入最终订单 |
| 退货 | 到货后客户退回，管理员在结算阶段标记 |
| 数量调整 | 管理员在结算阶段根据实际情况修改数量 |

### 5.3 完整流转图

```

                    │

                    │

                    │
                    ↓

                    │
              ┌─────┴──────┐
              ↓            ↓
         付定金        一次付清
              │            │
              ↓            ↓
        【部分付款】   【已付清】
              │            │
        ┌─────┴─────┐      │
        ↓           ↓      │

        │           │      │
```

### 5.4 定价、汇率与加价机制

本平台存在**三层定价逻辑**，是核心业务机制：

| 层级 | 币种 | 角色 | 说明 |
|------|------|------|------|
| 成本价 | **CNY** | 管理员 | 管理员为每个 SKU 填写的人民币采购成本价 |
| 加价比例 | 倍数 | 管理员 | 每个客户有独立的加价比例（默认 1.15，即加价 15%） |
| 客户价 | **SAR** | 客户 | 客户看到的价格 = 成本价 × 加价比例 × 汇率 |

**客户加价比例规则：**

- 每个客户有独立的 `markup_ratio` 字段（默认 1.15）
- 管理员审核客户注册时**必须**填写加价比例（默认预填 1.15）
- 管理员可随时在客户管理中修改某个客户的加价比例
- 客户浏览商品时，看到的参考价 = SPU 参考价 × 该客户的加价比例
- 未登录或未审核通过的用户看不到任何价格

**订单报价流程：**

```
管理员为每个订单项填写 CNY 成本价
              ↓
管理员确认/调整本次汇率（如 1 CNY = 0.51 SAR）
管理员确认/调整本次加价比例（默认取该客户的 markup_ratio）
              ↓
系统自动计算：SAR 价格 = CNY 成本价 × 加价比例 × 汇率
              （结果保留小数点后1位，向上取整，如 45.123 → 45.2）
              ↓
管理员预览：
  - 成本总价（CNY）
  - 客户总价（SAR）
  - 使用的汇率和加价比例
  - 预估毛利
              ↓
确认提交 → 订单锁定汇率和加价比例
              ↓
客户收到报价全部以 SAR 展示
（客户不可见：成本价、加价比例、毛利信息）
```

**付款前可调整：**

管理员可在客户确认订单但未全额付款前，一次性修改：
- 汇率（系统重算全部订单项的 SAR 价格）
- 加价比例（系统重算全部订单项的 SAR 价格）
- 或直接修改订单总金额（覆盖计算结果，用于对齐线下协商）

**订单需记录的定价信息：**

| 字段 | 说明 |
|------|------|
| exchange_rate | 本次报价使用的汇率（CNY→SAR） |
| markup_ratio | 本次报价使用的加价比例 |
| quoted_at | 报价确认时间 |
| total_cny | 成本总价（人民币） |
| total_sar | 客户总价（沙特里亚尔） |
| override_total_sar | 管理员手动覆盖的总金额（如有，为空则使用计算值） |

> 每次重新报价（协商后）可使用新的汇率和加价比例。历史记录随订单保存，不会被覆盖。

### 5.5 付款记录

支持分期付款，每笔付款记录：

| 字段 | 说明 |
|------|------|
| 付款金额 | 本次付款数额（SAR） |
| 付款方式 | 银行转账 / Western Union / 现金 / 其他 |
| 付款凭证 | 可选，客户或管理员上传截图 |
| 备注 | 说明（如"定金30%"） |
| 记录时间 | 管理员确认时间 |

### 5.6 物流信息

| 字段 | 说明 |
|------|------|
| 物流单号 | 如 COSCO2024031501 |
| 物流查询链接 | 如 https://www.track-trace.com/... |
| 运输方式 | 海运 / 空运 / 快递 |
| 备注 | 如预计到达时间等 |

### 5.7 结算机制

订单在满足（已发货 + 已付清）条件后进入「结算中」环节。这是为了处理到货后可能发生的退货、数量不符、额外折扣等现实情况。

**结算流程：**

```
订单已发货 + 已付清
         ↓
    进入【结算中】
         ↓
  管理员根据到货实际情况操作：
    - 标记退货商品（订单项状态改为"退货"）
    - 调整商品数量（记录结算数量）
    - 调整单价或给予额外折扣
    - 填写结算备注
         ↓
  系统自动计算：
    - 结算总额（CNY / SAR）
    - 与原报价总额的差异
    - 应退/应补金额
         ↓
  管理员确认结算
         ↓
    进入【已完成】
```

**结算对比明细（管理员和客户双方可见）：**

| 项目 | 说明 |
|------|------|
| 报价总额 | 原订单确认时的总金额（SAR） |
| 结算总额 | 到货后最终确认的金额（SAR） |
| 差异金额 | 结算总额 - 报价总额（正数为应补，负数为应退） |
| 结算备注 | 管理员填写的说明（如"客户退回3件戒指，给予5%折扣"） |

> 如果到货无任何异议，管理员可快速确认结算（不做调整，直接完结），此时结算总额 = 报价总额。

### 5.8 运费与毛利

**运费：**
- 管理员可为每个订单手动输入运费（CNY）
- 运费在发货时或之前输入，用于毛利计算

**毛利计算（仅管理员可见）：**

| 指标 | 计算公式 | 时机 |
|------|---------|------|
| 预估毛利 | 成本总价(CNY) × (加价比例 - 1) - 运费(CNY) | 订单完结前 |
| 实际毛利 | 结算总额(CNY等价) - 成本总价(CNY) - 运费(CNY) | 订单完结后 |

> 其中：成本总价 = sum(各订单项 CNY 成本价 × 数量)。毛利信息**不向客户展示**，仅管理员在订单详情和订单列表中可见。

## 九、设计原则

### 9.1 品牌视觉

- **主色调**: 黑色 + 金色（与 Logo 一致）
- **辅助色**: 深灰、暖白
- **字体**: 优雅衬线体（Logo）+ 清晰无衬线体（正文）
- **风格**: 高端珠宝感，大量留白，注重图片展示

### 9.2 客户端设计（Mobile-first）

- 底部 Tab 导航（商品 / 购物车 / 订单 / 我的）
- 大图展示商品，卡片式布局
- 触控友好的操作区域（最小 44px）
- 离线时的优雅降级提示

### 9.3 管理后台设计（Desktop-first）

- 左侧边栏导航
- 数据表格为主的操作界面
- 移动端可查看但不作为主要操作端
