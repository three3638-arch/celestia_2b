# 商品分组权限管理功能

## 需求概要

- 商品可归属一个分组，分组可动态创建
- 管理员控制客户可见的分组（多选），仅影响未来可见性
- 无分组商品对所有客户不可见
- 商品列表支持批量设置分组和分组筛选（含"无分组"选项）
- 商品编辑时可编辑/新建分组
- 客户端对分组无感知

## Task 1: 数据库模型扩展

**文件**: `prisma/schema.prisma`

新增两个模型：

```prisma
model ProductGroup {
  id        String   @id @default(cuid())
  name      String   @unique
  createdAt DateTime @default(now()) @map("created_at")
  updatedAt DateTime @updatedAt @map("updated_at")

  products  Product[]
  users     UserGroupAccess[]

  @@map("product_groups")
}

model UserGroupAccess {
  id        String   @id @default(cuid())
  userId    String   @map("user_id")
  groupId   String   @map("group_id")
  createdAt DateTime @default(now()) @map("created_at")

  user  User         @relation(fields: [userId], references: [id], onDelete: Cascade)
  group ProductGroup @relation(fields: [groupId], references: [id], onDelete: Cascade)

  @@unique([userId, groupId])
  @@map("user_group_access")
}
```

修改 `Product` 模型，新增：
```prisma
groupId      String?       @map("group_id")
group        ProductGroup? @relation(fields: [groupId], references: [id], onDelete: SetNull)
```

修改 `User` 模型，新增：
```prisma
groupAccess  UserGroupAccess[]
```

生成并执行 migration。

## Task 8: 前台商品过滤逻辑

**修改文件**: `src/lib/actions/product.ts` (前台 `getProducts`)

核心逻辑：
```typescript
// 1. 获取当前用户的可见分组
const accessibleGroupIds = await prisma.userGroupAccess.findMany({
  where: { userId: user.id },
  select: { groupId: true }
})

// 2. 商品查询条件增加：
// - groupId 不为 null（无分组商品不可见）
// - groupId 在用户可见分组列表中
where: {
  ...existingFilters,
  groupId: { in: accessibleGroupIds.map(g => g.groupId) }
}
```

同步修改前台的收藏列表 `getFavorites()`、隐藏列表 `getHiddenProducts()` 中的商品查询，确保也受分组权限约束。

注意：前台 UI 不展示任何分组相关信息，用户无感知。

## 风险与注意事项

1. **数据迁移**：现有商品 `groupId` 默认为 null，即所有现有商品变为"无分组"状态。上线前需提醒管理员批量设置分组，否则客户将看不到任何商品。建议在上线部署后，管理员立即使用"一键分配无分组商品"功能。
2. **历史订单**：订单项存储了商品快照（`productNameSnapshot`、`skuDescSnapshot`），分组变更不影响历史订单数据。
3. **性能**：前台商品查询增加了 `groupId IN (...)` 条件，需确保 `groupId` 字段有索引（Prisma 外键自动创建索引）。
4. **product.ts 并发编辑风险**：Task 2/5/6/8 都涉及修改 `src/lib/actions/product.ts`，需合并为同一 Coding Agent 或严格串行处理。
