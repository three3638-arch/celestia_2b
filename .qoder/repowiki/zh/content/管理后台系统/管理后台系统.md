## 简介

Celestia管理后台系统是一个基于Next.js 16构建的专业珠宝管理平台，专为Celestia珠宝品牌设计。该系统提供了完整的后台管理功能，包括商品管理、订单处理、客户关系管理和系统设置等核心业务模块。

系统采用现代化的技术栈，结合了TypeScript类型安全、Prisma ORM数据持久化、JWT认证机制和响应式UI设计，为企业级珠宝管理提供了强大而灵活的解决方案。

**更新** 管理后台界面已简化，移除了settings导航项，专注于核心业务功能。新增Excel导入导出功能，支持批量商品数据处理和模板下载。**新增订单已删除商品处理优化**：在管理员订单详情和报价页面中增强了已删除商品的显示和管理功能，包括智能排序、视觉指示和状态管理，提升了订单管理的透明度和用户体验。**新增1688导入和同步功能**：系统集成了阿里巴巴1688开放平台API，支持从1688批量获取商品数据、自动属性映射和价格同步，大幅提升了商品数据管理效率。

### 管理布局组件

AdminLayout组件是整个管理后台的核心布局组件，实现了响应式设计和完整的导航系统。

#### 组件特性

- **响应式设计**：支持桌面端和移动端的自适应布局
- **动态导航**：根据当前路径高亮显示活动菜单项
- **权限控制**：集成JWT认证和角色权限验证
- **主题设计**：采用深色主题配色方案，符合珠宝品牌的高端定位

系统提供四个主要功能模块（已移除settings导航项）：

| 模块 | 路径 | 功能描述 |

#### 商品管理功能

#### 已删除商品处理优化

**更新** 管理后台新增了已删除商品处理优化功能，显著提升了订单管理的透明度和用户体验：

##### 订单详情页面优化

在管理员订单详情页面中，已删除商品（CUSTOMER_REMOVED状态）得到了专门的处理：

1. **智能排序**：已删除商品会被自动移动到列表末尾，确保当前有效商品优先显示
2. **视觉指示**：已删除商品行具有半透明效果（opacity-50）和删除线样式（line-through）
3. **状态标识**：显示"已删除"状态标签，使用灰色背景和边框
4. **计算过滤**：在计算总成本和总售价时，已删除商品会被自动排除

##### 报价页面优化

在管理员报价页面中，已删除商品同样得到了优化处理：

1. **排序规则**：已删除商品被移动到列表末尾，便于管理员识别
2. **视觉样式**：已删除商品行具有半透明效果和删除线
3. **交互限制**：已删除商品的输入字段被禁用，防止误操作
4. **价格计算**：报价计算时自动忽略已删除商品

##### 状态管理机制

系统中的CUSTOMER_REMOVED状态具有以下特点：

- **非物理删除**：商品记录仍然存在于数据库中，但标记为已删除状态
- **历史追踪**：保留完整的购买历史和价格信息
- **权限控制**：只有管理员可以看到已删除商品的详细信息
- **计算隔离**：在财务计算中自动排除已删除商品

### 1688导入对话框

Ali1688ImportDialog提供了完整的1688商品导入流程，包含四个主要步骤：

#### 步骤1：输入参数
- **供应商ID**：输入1688卖家OpenID
- **供应商名称**：输入供应商公司名称
- **SPU前缀**：设置商品SPU编码前缀
- **汇率设置**：配置CNY到SAR的汇率
- **获取数量**：设置从1688获取的商品数量上限

#### 步骤2：获取商品数据
系统通过querySellerOfferList API分页获取1688商品清单，然后并发调用queryProductDetail获取详细信息。支持最多5个并发请求，提升获取效率。

#### 步骤3：属性映射确认
系统自动收集所有唯一的SKU属性名，并提供下拉选择框让用户将1688属性映射到系统内部属性：
- 宝石类型（gemType）
- 金属底色（metalColor）
- 主石尺寸（mainStoneSize）
- 尺码（size）
- 链长度（chainLength）
- 忽略（ignore）

#### 步骤4：入库结果
导入完成后显示详细的导入统计信息，包括成功导入的商品数量、SKU数量，以及导入失败的商品列表。

### 1688同步对话框

Ali1688SyncDialog专门用于同步1688商品的价格和状态：

#### 同步流程
1. **权限验证**：确保只有管理员可以执行同步操作
2. **查询数据库**：获取指定供应商的所有商品（含SKU）
3. **获取1688清单**：分页获取当前在售商品清单
4. **对比处理**：
   - 缺失商品：自动下架（设置为INACTIVE）
   - 新增商品：无需处理（等待下次导入）
   - 存在商品：更新价格和库存状态
5. **统计结果**：显示同步统计信息

#### 同步统计
- **总商品数**：该供应商在系统中的商品总数
- **价格更新**：成功更新价格的商品数量
- **已下架**：因1688下架而被下架的商品数量
- **SKU缺货**：因1688下架而标记为缺货的SKU数量
- **处理失败**：同步过程中失败的商品数量

## 附录

### 开发者指南

#### 环境要求
- Node.js 16+
- PostgreSQL数据库
- Prisma CLI（可选）
- 1688开放平台API密钥

#### 本地开发步骤
1. 克隆项目到本地
2. 安装依赖：`npm install`
3. 配置数据库连接
4. 配置1688 API环境变量
5. 运行开发服务器：`npm run dev`

#### 1688 API配置
1. 在`.env.local`文件中添加1688配置：
   ```
   ALI_1688_APP_KEY=your_app_key
   ALI_1688_APP_SECRET=your_app_secret
   ALI_1688_ACCESS_TOKEN=your_access_token
   ALI_1688_API_HOST=gw.open.1688.com
   ```

#### 部署建议
1. 使用生产环境配置
2. 配置JWT密钥和数据库连接
3. 设置反向代理和SSL证书
4. 配置监控和日志记录
5. 配置1688 API密钥和网络访问

### 管理员操作手册

#### 登录流程
1. 访问管理后台登录页面
2. 输入手机号和密码
3. 点击登录按钮
4. 成功登录后进入仪表盘

#### 基本操作
1. **商品管理**：添加新商品、编辑现有商品信息、管理库存、批量导入Excel数据、导入1688商品数据
2. **订单处理**：查看订单详情、更新订单状态、处理客户咨询、删除订单
3. **客户管理**：查看客户信息、处理客户申请、修改客户加价比例、分析客户行为

#### Excel导入操作
1. **下载模板**：点击"下载模板"按钮获取Excel模板
2. **填写数据**：按照模板格式填写商品信息，包括新增的主石尺寸字段
3. **上传文件**：点击"Excel导入"按钮上传文件
4. **确认导入**：系统验证数据后进行批量导入

#### SKU编辑操作
1. **添加SKU**：点击"添加SKU"按钮创建新的规格组合
2. **编辑规格**：通过下拉菜单选择宝石类型、金属底色等规格
3. **设置主石尺寸**：在主石尺寸字段输入具体的尺寸数值（mm）
4. **管理库存**：设置每个SKU的库存状态和参考价格
5. **调整顺序**：使用上下箭头按钮调整SKU的显示顺序

#### 已删除商品处理操作
1. **识别已删除商品**：在订单详情和报价页面中，已删除商品会显示"已删除"状态标签
2. **理解视觉效果**：已删除商品行具有半透明效果和删除线样式
3. **查看历史信息**：管理员可以查看已删除商品的完整历史信息
4. **财务计算**：系统会自动排除已删除商品进行财务计算
5. **恢复操作**：如需恢复已删除商品，可通过相应的管理功能进行操作

#### 1688导入操作
1. **打开导入对话框**：在商品管理页面点击"从1688导入"按钮
2. **输入供应商信息**：填写供应商ID、供应商名称、SPU前缀、汇率和获取数量
3. **开始获取**：点击"开始获取"按钮从1688获取商品数据
4. **属性映射**：在属性映射页面为1688属性选择对应的系统属性
5. **确认导入**：点击"确认并入库"完成商品导入
6. **查看结果**：导入完成后显示详细的导入统计信息

#### 1688同步操作
1. **打开同步对话框**：在商品管理页面点击"同步1688商品"按钮
2. **输入供应商信息**：填写供应商ID和现行汇率
3. **开始同步**：点击"开始同步"按钮执行价格和状态同步
4. **查看统计**：同步完成后显示详细的同步统计信息
5. **处理结果**：根据统计结果进行相应的商品管理操作

#### 数据安全
- 所有敏感数据传输均通过HTTPS加密
- 用户密码采用bcrypt加密存储
- 管理员操作均有审计日志记录
- 定期备份数据库和重要文件
- Excel文件上传进行类型和大小验证
- 1688 API调用通过专用模块封装，确保安全性
